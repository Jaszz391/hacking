Kompendium Lengkap (ringkasan topik yang dimasukkan)

Dokumen ini sekarang berisi: glossary ekstensif, threat actor taxonomy, social engineering, phishing patterns (deskriptif), OSINT overview, wireless & IoT security, cloud & container security concepts, cryptography basics, malware classification (deskripsi), ransomware overview, DDoS overview, supply-chain risks, mobile security principles, red-team vs blue-team roles, SIEM & detection engineering, hunting techniques (conceptual), legal frameworks, compliance (GDPR, PCI-DSS ringkasan), bug bounty processes, responsible disclosure templates, policy templates (password, patching, access), checklist & playbooks, forensic guidance, and learning roadmap.

Threat actors & motivations

  • Script kiddies — opportunistic, low-skill.
  • Cybercriminals — financial gain, ransomware.
  • Hacktivists — political/ideological motives (deface common).
  • State-sponsored actors — espionage, targeted campaigns.
  • Insider threats — negligence or malicious insiders.

Social engineering & phishing (descriptive)

Social engineering exploits human factors. Phishing campaigns vary: mass phishing, spear-phishing (targeted), vishing (voice), smishing (SMS). Mitigations: security awareness training, email filtering, DMARC/SPF/DKIM, simulated phishing tests.

OSINT (Open Source Intelligence) — penggunaan & mitigasi

OSINT is collecting publicly available information (LinkedIn, GitHub, social media, domain WHOIS). Useful for defender threat modeling and attacker reconnaissance. Mitigation: reduce public exposure (remove sensitive docs, harden DNS), monitor brand mentions.

Wireless, IoT & Embedded Systems (overview)

IoT devices often lack secure defaults. Focus: network segmentation, regular firmware updates, strong device authentication, disable unused services, monitor device traffic.

Cloud & container security concepts

  • Least privilege IAM (use roles, avoid long-lived keys).
  • Secure images — scan base images, sign images.
  • Network segmentation (VPCs, security groups).
  • Secrets management (KMS, Vault).
  • Runtime security: container scanning, host hardening.

Cryptography basics (non-math)

Symmetric vs asymmetric, hashing vs encryption, key management, TLS basics, certificate management, perfect forward secrecy concept. Emphasize correct usage and key lifecycle.

Malware taxonomy & ransomware (descriptive)

Categories: trojans, worms, backdoors, rootkits, ransomware. Ransomware: encryption of data for extortion; mitigations: backups, network segmentation, IR plan, threat intel.

DDoS overview

DDoS floods service with traffic/requests. Mitigations: CDN/WAF, rate limiting, autoscaling, scrubbing services, traffic filtering.

Supply chain & dependency risks

Supply chain attacks target third-party components. Best practices: SBOM, pin dependencies, monitor CVEs, approve changes, use signed releases.

Mobile security principles

App hardening, secure storage of secrets, avoid logging sensitive data, apply platform best practices (Android/iOS), update lifecycle, code obfuscation if appropriate, use platform-provided secure keystores.

Detection engineering & hunting (conceptual)

Create detections for anomalous patterns: unusual admin login time, new webshell indicators, mass file changes, DNS anomalies. Use baselining, threat intel feeds, and tuning to reduce false positives.

Legal & compliance overview

High-level notes on GDPR, PCI-DSS, local cybercrime laws: maintain data protection, breach notification timelines, and contractual security controls. Always consult legal counsel for specifics.

Red Team / Blue Team roles

Red team (adversary emulation) vs blue team (defense). Purple team = collaboration to improve detection. Emphasize authorized exercises, scope, and rules of engagement.

Responsible disclosure & bug bounty

How to set up a program: scope, reward tiers, contact channel, triage process, legal safe harbor. Encourage external researchers to report responsibly.

Quick actions

Printable checklist

  • Enable HTTPS & HSTS
  • Rotate keys & MFA
  • Patch & scan weekly
  • Backup & test restores
  • Implement WAF & logging
